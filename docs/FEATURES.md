## 导入 KML

- **入口**：右上角 `➕ 添加轨迹` 按钮 或直接拖入浏览器窗口
- **多选**：一次可选多个 KML 文件
- **解析**：
  - 合并同一 KML 中的 `<gx:Track>` 与 `<LineString>` 轨迹段（按 `<name>` 的数字序号排序）
  - `<Point>` 类型 placemark 识别为标注点
  - 坐标顺序 `(lon, lat, alt)` → Leaflet `(lat, lon)` 自动翻转
- **计算**：导入即时算距离/爬升/降/最高/最低/分天

## 多轨迹叠加与主轨迹

- 所有轨迹默认叠加在同一张地图上
- **主轨迹**：粗线 + 全不透明 + 显示所有标注点
- **其他轨迹**：`opacity: 0.45`, `weight: 2.5`，只有轨迹线（不显示标注点，避免重叠混乱）
- 点击轨迹卡片的 `⭐ 设为主轨迹` 一键切换
- 数据 `DATA.trails` 数组第一个默认为主轨迹

## 拼接已有轨迹

- **入口**：顶部 `编辑` → `拼接轨迹`
- 来源选择初始为 0；选择至少两条已有轨迹后进入专用地图拼接工作台
- 每个片段在地图上显示可拖动的 A/B 点，只吸附其来源轨迹；可恢复全段、复制或删除片段
- 片段列表支持拖放或按钮排序，并可单独反向任意片段；方向箭头和颜色与地图预览一致
- 5 m 内的重复接缝点自动合并；更远的端点记录为断点，只显示编辑辅助虚线，最终地图不画连接线
- 断点距离和高差不进入里程、爬升或下降，海拔图、测距和 Day 预览也会在断点处断开
- 生成结果是一条独立的新轨迹，原轨迹不会被修改
- 新轨迹会重新计算里程、爬升、下降、最高和最低海拔，并将来源标注点重映射到新索引
- KML 导出以多个 `LineString` 保存断开的路线部分，避免其他地图软件绘制跨空白区域的直线
- 日程不会从来源轨迹混合继承；需要时使用 `规划` → `行程分段` 为新轨迹重新规划

## 三种着色模式

顶部工具栏「模式」下拉切换：

### 天数模式
- 按 GPS 时间戳北京时间 `00:00` 切分日期
- 每天一种颜色（10 色调色板循环）
- 侧栏 legend 显示每天里程/爬升/时长

### 海拔模式
- 按海拔高度做梯度：低海拔绿 → 高海拔红
- 使用 5 段渐变（低/较低/中/较高/最高）
- 适合看穿越海拔起伏

### 标注点对比模式
- 只有主轨迹显示轨迹线（淡）
- 所有轨迹的标注点按 tag 分类同时显示
- 便于对比不同用户在同一路线上的标注差异（如「A 说这里是营地，B 说这里是水源」）

## 标注点系统

### 13 类 tag（自动分类）

| Tag | 关键词 |
|-----|--------|
| `camp` | 营地、宿营、扎营、tent、camp |
| `pass` | 垭口、pass |
| `water` | 水源、河、湖、water、lake、river |
| `bridge` | 桥、bridge |
| `river` | 河流（与 water 稍有区别） |
| `supply` | 补给、supply、food |
| `start` | 起点、start、entry |
| `end` | 终点、end、exit |
| `pasture` | 牧场、pasture |
| `snow` | 雪、glacier |
| `pit` | 坑、pit |
| `viewpoint` | 观景、viewpoint |
| `other` | 其他 |

分类规则见 [references/tag-rules.md](../references/tag-rules.md)。

### 默认显示 tag

`camp`, `pass`, `water`, `bridge`, `river`, `supply`, `start`, `end` —— 徒步实用性最强的 8 类。

### 交互
- **筛选**：侧栏 chip 点击开/关
- **改名**：双击标注点弹出编辑框
- **手动添加**：
  - 桌面：地图上右键 → 弹菜单 → 选 tag / 输入名字
  - 手机：长按 600ms → 弹菜单
- **snap-to-track**：手动添加的标注点会自动投影到最近的 GPS 轨迹点上，视觉紧贴轨迹线

## 海拔图

底部固定海拔剖面图（Canvas 2D）。

### 展示内容
- X 轴：里程（km）
- Y 轴：海拔（m）
- 曲线：主轨迹的海拔剖面
- 标注：营地（🏕）+ 最高点 + 最低点，含引线连接曲线点与标签
- 分天：垂直虚线切分不同天数

### 关键设计
- **自适应高度**：扫描线预测算法计算 X 方向标签最大堆叠数，公式 `H = max(140, min(340, 110 + (stackDepth+1) × (lh+2) + 16))`
- **点击反向定位**：点海拔图任意位置 → 地图上高亮对应 GPS 点 + 浮标显示海拔/里程
- **km 映射一致性**：绘制和 hit-test 都按 km 而非 idx 映射 X，确保点击位置精确

## 轨迹分组 (v1.14.0+)

- **分组字段**：`trail.group`（默认 `"默认"`）
- **入口**：侧栏顶部 Tab 栏（>1 组时显示）
- **规则**：切到某组后，地图 / 海拔图 / 统计 / 图例只显示该组
- **移动**：展开轨迹卡片 → 详情下拉 → 选择目标组 或「＋ 新建组…」
- **新导入**：默认加入当前激活组

## 批量分组 (v1.14.1)

侧栏顶部「☐ 批量分组」按钮进入批量模式：

- **进入**：`已选 0/N` 提示 + 全选 · 反选 · 移到 · 退出
- **选择**：点击卡片任意区域切换选中态；选中卡片显示实线橙色描边
- **全选/反选**：一键操作
- **移到**：下拉选择目标组，或「＋ 新建组…」立即新建
- **执行**：一次搬运所有选中，Toast 提示搬了几条
- **主轨迹处理**：如果主轨迹被移走，自动把当前组第一个残留 trail 设为新主

## 导出

`📤 导出` 按钮点击后在按钮下方悬浮弹出选项菜单（v1.14.1 起）：

### 📦 打包 KML ZIP

- 内容：当前组所有「叠加中」的轨迹
- 结构：
  ```
  格聂北线_轨迹_2026-07-01.zip
  ├── 轨迹/
  │   ├── 张三_四天路线.kml
  │   ├── 李四_变体.kml
  │   └── ...
  ├── _格聂北线_合并导入.kml   ← 一键导入用（内嵌 <Folder>）
  └── README.txt
  ```
- 用途：跨设备迁移、发给队友
- 降级：fflate 不可用时逐条下载

### 📄 行程 Markdown

- 每日行程表：`天数 / 行程路段 / 徒步距离 / 当日爬升 / 最高海拔 / 扎营点(海拔) / 下撤方案`
- 标注点列表（含 `[手动]` 标记）
- 复制粘贴到 Notion / VS Code 都能保持格式

## 持久化

- **存储**：IndexedDB，key = `main`
- **写入策略**：300ms debounce（避免每次 UI 变化立即写盘）
- **恢复**：启动时 `_boot` 自动 load
- **持久化字段**：`trails`（含 tracks + waypoints）· `activeTrails` · `primaryTrailId` · `activeGroup`
- **清空**：浏览器清缓存 或 侧栏底部「🗑 清空数据」按钮

## 国际化

- 中文 / 英文双语，右上角 `🌐 中/EN` 切换
- 用 `data-i18n` 属性 + `t()` 函数管理翻译表
- 切语言后所有动态渲染都会 refresh（海拔图 / 主轨迹卡 / mode 标题等）

## 快捷键与手势

| 场景 | 操作 |
|------|------|
| 地图缩放 | 滚轮 · 双击 · 双指捏合 |
| 浏览器整页放大 | Cmd/Ctrl + / − / 0（不会被地图吞掉）|
| 取消当前操作或关闭浮层 | Escape（通过统一 `interaction.cancel` 命令）|
| 添加标注点 | 桌面右键 / 手机长按 600ms |
| 改名 | 双击标注点 或 双击轨迹卡片标题 |
| 反向轨迹 | 展开轨迹详情 → `↔ 反向` 按钮 |
| 展开/折叠所有卡片 | 侧栏顶部快捷按钮 |
