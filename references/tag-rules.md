# 标注点自动分类规则

把用户原始标注的文本归到统一 tag 系统，用于地图筛选和图标显示。

## tag → 关键词映射（按优先级，先匹配的优先）

```python
TAG_RULES = [
    # (tag, [关键词列表], icon, color)
    ('start',   ['开始徒步', '起点', '出发', '起步'], '🚩', '#5eb3ff'),
    ('end',     ['终点', '结束', '收队', '到达终点'], '🏁', '#5eb3ff'),
    ('fork',    ['分叉', '分岔', '路口', '走左边', '走右边', '切下去', '右转', '左转', '岔路'], '⑫', '#ff8c42'),
    ('camp',    ['营地', '扎营', '宿营', '过夜', 'D1', 'D2', 'D3', 'D4', 'D5'], '🏕', '#22c55e'),
    ('pass',    ['垭口', '口子'], '🏔', '#ef4444'),
    ('warn',    ['Z字', '陡', '危险', '注意', '小心', '高反', '滚石', '滑'], '⚠', '#dc2626'),
    ('supply',  ['商店', '补给', '便宜', '柠檬茶', '咖啡', '卖', '夯达'], '🛒', '#facc15'),
    ('water',   ['水源', '打水', '取水'], '💧', '#3b82f6'),
    ('shelter', ['避雨', '避风', '小木屋', '木屋'], '🏠', '#a855f7'),
    ('bridge',  ['过桥', '过河', '涉水'], '🌉', '#06b6d4'),
    ('river',   ['小溪', '大河', '溪', '河'], '〰', '#06b6d4'),
    ('village', ['村', '寨', '牧民', '藏民', '居民点'], '🏘', '#d97706'),
    ('view',    ['神山', '雪山', '海子', '花海', '观景', '风景', '景点', '美', '漂亮', '看不腻', '寺'], '⛰', '#94a3b8'),
]
```

## 匹配代码

```python
def classify(name):
    for tag, keywords, icon, color in TAG_RULES:
        for kw in keywords:
            if kw in name:
                return tag, icon, color
    return 'view', '📍', '#aaa'   # 默认归入景点
```

## 特殊处理

- **未命名标注点**（name 为空）：默认归类为 `view`，但通常不显示在地图上
- **D1/D2/D3 标识**：通过日期前缀识别营地优先级
- **`补给` 和 `营地` 同时出现**（如"夯达营地（补给很便宜）"）：归类为 `supply` 但在地图上同时显示 🛒🏕

## 用户自定义

如果用户对某个标注点的分类有不同意见，可以在 `data/<trail-id>/overrides.json` 写：

```json
{
  "76": {"tag": "shelter", "icon": "🏠"},
  "50": {"tag": "bridge", "icon": "🌉"}
}
```

`analyze_trail.py` 处理时会优先用 overrides 里的值。
